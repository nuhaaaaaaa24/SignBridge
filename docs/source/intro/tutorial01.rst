=====================
User Journey Overview
=====================

This tutorial walks you through how to use SignBridge after installation and setup. 
It covers joining a session, interacting in a call room, and using sign language recognition.

Step 1: Log in
===============

Log in to the system. If you do not have an account, you can register using the 
**Don't have an account? Click here** link on the login page.

.. note::

   Make sure to allow camera and microphone permissions in your browser before joining a session.


Step 2: Access Session Page
===========================

Navigate to **Join Session** in your dashboard after logging in.

Step 3: Create or Join a Session
=================================

After you click on **Join session**, you will be directed to the session page where you can:

1. Click on **Create a new session** if you do not have a room code.
2. Click on **Join a session** if you already have a room code or if the session was created by another user.

.. tip::

   If you are creating the new session, share the generated room code to the other participant before the session starts.


Step 4: Enter Call Room
=======================

Once both users have joined the session, you will be redirected to the call room 
where real-time interaction takes place.

Step 5: Use Chat
=================

You can use the chat feature to communicate with the other participant during 
the session if needed.

Step 6: Start Sign Language Recognition
========================================

Click the **Start Recognition** button to begin detecting the sign language 
performed by the other user.

.. warning::

   Ensure you are in a well-lit environment for accurate sign detection.


Step 7: View Transcript Output
==============================

The recognized sign language letter will be displayed in the transcript in real time.

Troubleshooting
===============

**Camera not working**
   Check that your browser has permission to access the camera. Refresh the page and try again.

**Room code not working**
   Double-check the code with the session creator. Codes are case-sensitive.
